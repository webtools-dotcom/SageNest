## 🧠 PHILOSOPHY: WHAT ACTUALLY MAKES A PIN GO VIRAL

Before touching any design or copy, internalize this:

**Pinterest's algorithm rewards SAVES, not clicks.**  
A save happens when a user thinks: *"This is too useful to lose."*  
A click happens when a user thinks: *"I need to know more."*

Your pin must trigger BOTH — in that order.

The formula is:
```
Verified, specific, surprising information
→ User saves it (algorithm distributes it)
→ Curiosity gap at the bottom ("there's more")
→ User clicks through to blog
```

**What you were doing wrong before this guide:**
- Pins were teasers that needed the blog to make sense → low saves → algorithm buried them
- No verified facts, just generic reminders → no trust, no saves
- Design optimized for clicks, not saves → wrong optimization target

**The new rule:**  
Every pin must be valuable EVEN IF the user never clicks.  
That's what makes them save it. That's what triggers viral distribution.

## ⚠️ SECTION 1: ANTI-HALLUCINATION PROTOCOL — NON-NEGOTIABLE

This section must be applied BEFORE writing any copy. Skipping it is not allowed.

### 1A. TEXT ACCURACY RULES (Prevent AI Text Errors in Design)

When passing text to any image generation tool or Canva template:

**RULE 1 — Double-check every word character by character before finalizing**
Common AI text hallucination patterns to watch for:
- "about" → "aout", "abut", "abotu"
- "pregnancy" → "pregnany", "pregancy"
- "trimester" → "trimster", "trimaster"
- "nausea" → "nasea", "nauesa"
- "folic" → "foilc", "flocic"
- Numbers getting swapped: "400mcg" → "40mcg" (DANGEROUS — medical doses must be exact)

**RULE 2 — Never use decorative fonts for medical numbers, doses, or statistics**
Medical figures must be in clean, unambiguous typefaces.  
Example: "400mcg of folic acid" must be rendered in body font, not script/cursive.

**RULE 3 — All text on pin must be typed out explicitly in the Canva text layer**
Never rely on image generation to render text. All copy goes in Canva text boxes only.

**RULE 4 — Verify the pin text against the source URL one final time before export**
Read the pin text aloud against the source. If a single word is different, fix it.

**RULE 5 — No abbreviations that could be misread**
- "mg" is acceptable (universal)
- "IU" — write out "International Units" in parentheses next to it
- "OB/GYN" — acceptable
- Never use abbreviations not found in the original source

### 1B. SOURCE VERIFICATION PROTOCOL — MANDATORY BEFORE EVERY CLAIM

**Approved Sources (Tier 1 — Always prefer these):**
1. **ACOG** — acog.org (American College of Obstetricians and Gynecologists)
2. **WHO** — who.int (World Health Organization)
3. **CDC** — cdc.gov (Centers for Disease Control)
4. **NIH / NCBI** — nih.gov / ncbi.nlm.nih.gov / PubMed
5. **IOM / NAM** — nationalacademies.org (Institute of Medicine / National Academies)
6. **Mayo Clinic** — mayoclinic.org
7. **WomensHealth.gov (OWH)** — womenshealth.gov
8. **FDA Pregnancy** — fda.gov (pregnancy-specific sections only)

**Approved Sources (Tier 2 — Use only if Tier 1 unavailable):**
- Cleveland Clinic — clevelandclinic.org
- Johns Hopkins Medicine — hopkinsmedicine.org
- Stanford Medicine — stanfordmedicine.org

**Banned Sources:**
- BabyCenter, What to Expect, Healthline, Verywell Family, mommy blogs
- Any source that does not cite a Tier 1 source for medical claims
- Reddit, forums, YouTube
- Any blog post (including SageNest itself as a primary source)

**Verification Workflow:**
```
Step 1: Read the blog post to identify the 5-7 key facts/claims
Step 2: For each claim, search the Tier 1 source directly
Step 3: Find the exact page/article that confirms it
Step 4: Copy the URL of that source page into your working notes
Step 5: Only write the claim in the pin if you found it on a Tier 1 source
Step 6: If a claim cannot be verified on Tier 1, DROP IT — do not include it
```

**Example of a VERIFIED claim (use this):**
> "Folic acid 400mcg daily before and during early pregnancy reduces neural tube defects by up to 70%."  
> ✅ Source: CDC — cdc.gov/folicacid

**Example of an UNVERIFIED claim (never use this):**
> "Eating ginger helps with morning sickness."  
> ❌ This is commonly repeated but needs a specific Tier 1 source confirming dosage/efficacy before using.

## 📋 SECTION 2: BLOG ANALYSIS PROTOCOL

When you receive a blog post file, do this before writing any pin copy:

### Step 1 — Extract the Core Topic
Answer these questions from the blog:
- What is the main topic in one sentence?
- What trimester or pregnancy stage is it about?
- What is the #1 problem this blog helps solve?
- What are the 5-10 most specific, actionable facts in the blog?

### Step 2 — Identify Pin-Worthy Claims
From the extracted facts, identify which ones are:
- **Specific** (has a number, timeframe, or measurable outcome)
- **Surprising** (most people don't already know this)
- **Actionable** (reader can do something with it today)
- **Verifiable** (can be confirmed on a Tier 1 source)

Rank them. Only the top 5-7 make it into a pin.

### Step 3 — Verify Each Claim (follow Section 1B protocol exactly)

### Step 4 — Identify the Curiosity Gap
Find 1-2 things in the blog that are genuinely interesting but would take too long to explain in a pin. These become your CTA hook. The pin gives the "what", the blog gives the "why" and "how".

## 🏗️ SECTION 3: PIN STRUCTURE TEMPLATES

Use one of these three structures depending on the blog topic. Do not invent new structures.

---

### STRUCTURE A: "N Things You Should Know" (List Format)
**Best for:** Tips, hacks, symptoms, foods, exercises  
**Target saves:** Highest — most shareable format

```
[HEADLINE]
[Number] [Topic] Every Pregnant Mom Should Know

[BODY — 5 to 7 items, each 1-2 lines max]
① [Bold Claim] — [One sentence explanation. Source verified.]
② [Bold Claim] — [One sentence explanation. Source verified.]
③ [Bold Claim] — [One sentence explanation. Source verified.]
④ [Bold Claim] — [One sentence explanation. Source verified.]
⑤ [Bold Claim] — [One sentence explanation. Source verified.]

[CURIOSITY GAP LINE]
Most moms skip #[X] — and it matters more than you think.

[CTA]
👉 Full breakdown with trimester timing at SageNest Health ↓
[Blog URL]
```

**Rules for this structure:**
- Headline number must match exact item count
- Each item: bold the claim, normal text for explanation
- Curiosity gap line must be honest — point to an item that actually has a deeper reason
- CTA never says "click here" — it says what they'll GET by going to the blog

---

### STRUCTURE B: "Did You Know?" (Single Stat Format)
**Best for:** Surprising single facts, statistics, research findings  
**Target saves:** Medium — works well for re-shares

```
[LARGE HOOK]
Did you know?

[THE STAT — large, bold, impossible to miss]
[Specific number or percentage or timeframe]

[BRIEF CONTEXT — 2 sentences max]
[What the stat means in plain language.]
[Why it matters for pregnancy.]

[SOURCE BADGE — small, bottom corner]
Source: [ACOG / CDC / WHO / Mayo Clinic]

[CURIOSITY GAP]
What you can do about this changes by trimester →

[CTA]
👉 See the full trimester guide at SageNest Health ↓
[Blog URL]
```

**Rules for this structure:**
- The stat must come directly from a Tier 1 source — no paraphrasing that changes the number
- Source badge is mandatory — it builds trust and reduces bounce
- "Changes by trimester" is the hook that drives clicks — only use this if the blog actually covers trimester differences

---

### STRUCTURE C: "Warning / Watch Out" (Alert Format)
**Best for:** Safety topics, things to avoid, symptoms to watch  
**Target saves:** High — fear-of-missing-out trigger, people save for safety

```
[HEADLINE]
[Number] [Topic] Signs During Pregnancy You Shouldn't Ignore

[BODY — 4 to 6 warning signs, each verified]
⚠ [Symptom/Sign] — [One line: why it matters, when to act]
⚠ [Symptom/Sign] — [One line: why it matters, when to act]
⚠ [Symptom/Sign] — [One line: why it matters, when to act]
⚠ [Symptom/Sign] — [One line: why it matters, when to act]

[IMPORTANT FOOTER — mandatory for any medical warning content]
Always consult your OB/GYN or midwife. This is educational only.

[CURIOSITY GAP]
Sign #[X] is often dismissed — here's what it can mean.

[CTA]
👉 Full symptom guide with what to tell your doctor at SageNest Health ↓
[Blog URL]
```

**Rules for this structure:**
- Disclaimer footer is NON-NEGOTIABLE on any warning/symptom pin
- Never say "this means you have X condition" — say "this may indicate" or "can be a sign of"
- Never cause unnecessary panic — pair each warning with one actionable next step

## 🎨 SECTION 4: DESIGN SPECIFICATION

Apply this design spec to every pin without exception. This is the SageNest visual identity.

### Canvas Setup
```
Dimensions: 1000 x 1500 px (2:3 ratio — Pinterest optimal)
Resolution: 300 DPI for export
Format: PNG (not JPEG — PNG preserves text sharpness)
```

### Color Palette — EXACT VALUES
```
Background:     #FAF7F2  (soft warm cream — never pure white)
Primary Text:   #2C2420  (warm dark brown — never pure black)
Accent Green:   #7B9E87  (muted sage green — headers, dividers, icons)
Highlight Warm: #D4956A  (soft terracotta — use sparingly, 1-2 elements max)
Divider Lines:  #E8E0D5  (barely-there warm grey)
Source Badge:   #F0EBE3  (slightly darker cream for badge background)
```

### Typography Rules
```
HEADLINE FONT:    Playfair Display Bold OR Cormorant Garamond Bold (serif editorial)
BODY FONT:        DM Sans Regular or Jost Regular (clean, readable sans-serif)
ACCENT/LABEL:     DM Sans Medium (for numbered items, category labels)

HEADLINE SIZE:    72-88px (must be readable in Pinterest feed thumbnail)
SUBHEADER SIZE:   36-42px
BODY TEXT SIZE:   28-32px (minimum — smaller is unreadable on mobile)
CTA TEXT SIZE:    26-30px
SOURCE BADGE:     20-22px (small but legible)
```

### Layout Rules
```
Top padding:      80px
Side padding:     60px
Bottom padding:   100px (CTA needs breathing room)

Between items:    32-40px gap
Divider lines:    1px, color #E8E0D5, full width minus side padding

Headline:         Centered or left-aligned (pick one, stay consistent)
Body items:       Left-aligned always
CTA block:        Centered, with subtle rounded-corner box around it
```

### Visual Elements
```
Item Markers:    Circled numbers ①②③ in accent green, or leaf/dot bullets
Dividers:        Thin horizontal lines between each item
Icon Style:      Minimal line icons, sage green, 48-64px — optional, not mandatory
Logo/Brand:      "SageNest Health" wordmark bottom center, small
Source Badge:    Bottom-left corner, pill shape, source name only
```

### What NOT to Do — Design Anti-Patterns
- ❌ No dark backgrounds (kills the warm editorial feel)
- ❌ No neon or saturated colors anywhere
- ❌ No more than 2 fonts on one pin
- ❌ No stock photo backgrounds (text becomes unreadable)
- ❌ No drop shadows on text (looks amateur)
- ❌ No centered body text (only headline can be centered)
- ❌ No font smaller than 24px on any element
- ❌ No more than 3 colors used visibly (background, text, one accent)

## ✍️ SECTION 5: COPYWRITING RULES

### Headline Formula
Your headline must do three things in order:
1. State exactly who it's for (pregnant moms, first trimester, etc.)
2. Promise specific value (not "tips" — "7 things", "the real reason", "signs")
3. Create mild urgency or curiosity without clickbait

**Good headlines:**
- "7 Pregnancy Nutrition Facts Most Doctors Don't Have Time to Explain"
- "5 Second Trimester Changes That Catch New Moms Off Guard"
- "What Your Iron Levels During Pregnancy Actually Mean"
- "6 Things to Know About Folic Acid Before Week 12"

**Bad headlines (never use these):**
- "Amazing Tips for Pregnancy!" (vague, no value promise)
- "You Won't Believe What Happens in Your Third Trimester" (clickbait, Pinterest penalizes)
- "Everything About Pregnancy Nutrition" (too broad, no specificity)

### Body Copy Rules
- Every item: **Bold claim first, then one-sentence explanation**
- Explanation: plain language, no medical jargon unless defined
- Max 18 words per explanation line
- Use real numbers when available: "400mcg", "28 weeks", "2x more likely"
- Never write "studies show" without specifying which study or organization

### Curiosity Gap Rules
The curiosity gap line is the bridge between the pin and the blog.  
It must be honest — if you tease something, the blog must actually cover it.

**Good curiosity gaps:**
- "Most moms are surprised by #4 — and it's the most important one."
- "The timing of #3 matters more than people realize — full breakdown in the blog."
- "One of these is controversial among OBs — the blog explains both sides."

**Bad curiosity gaps (never use):**
- "Click to find out more!" (too weak, no specific hook)
- "There's so much more to learn!" (vague)
- "This will change everything!" (overdramatic, untrustworthy)

### CTA Copy Rules
The CTA must tell users exactly what they get by going to the blog.

**Good CTAs:**
- "👉 Full trimester-by-trimester breakdown at SageNest Health ↓"
- "👉 Complete guide with what to tell your doctor — SageNest Health ↓"
- "👉 Week-by-week detail at sagenesthealth.com ↓"

**Bad CTAs:**
- "Visit our website!" (no value proposition)
- "Click here!" (Pinterest users don't respond to this)
- "Read more" (too weak)

## 🔍 SECTION 6: WEB SEARCH PROTOCOL FOR FACT-FINDING

When given a blog post, Claude must do the following search sequence:

### Search Step 1 — Topic Verification Search
Search: `[blog topic] site:acog.org`  
OR: `[blog topic] site:cdc.gov`  
Goal: Find the primary Tier 1 page that covers this topic

### Search Step 2 — Statistic Mining
Search: `[blog topic] statistics percentage [year] site:nih.gov`  
Goal: Find one surprising, specific stat that most people don't know

### Search Step 3 — Claim Confirmation
For each claim extracted from the blog:  
Search: `"[exact claim keyword]" site:mayoclinic.org OR site:acog.org OR site:who.int`  
Goal: Confirm the claim appears on a Tier 1 source before including it

### Search Step 4 — Competitor Pin Analysis (Optional but valuable)
Search: `[blog topic] pregnancy tips pinterest`  
Goal: See what's already ranking, then make sure your pin is MORE specific and MORE informative

## 🚫 SECTION 8: ABSOLUTE PROHIBITIONS

These are hard stops. If any of these apply, the pin cannot be published.

1. **Never make a diagnostic claim.** "This symptom means you have X" — prohibited.
2. **Never recommend a specific medication by brand name.** Generic nutrient names only (folic acid, iron, vitamin D).
3. **Never give dosage advice beyond what Tier 1 sources explicitly state for general pregnancy.**
4. **Never use before/after language.** No "this fixed my morning sickness."
5. **Never fabricate a statistic.** If you cannot find it on a Tier 1 source, it does not go on the pin.
6. **Never use fear-mongering headlines.** "This could KILL your baby" — never.
7. **Never imply SageNest is a medical authority.** Always position as educational resource.
8. **Never use testimonial language.** No "moms say", "many women report" unless citing a specific Tier 1 study.

## 🔁 SECTION 9: FULL GENERATION WORKFLOW (Step-by-Step)

When you receive a blog post, follow these steps in exact order:

```
STEP 1 — Read the full blog post
STEP 2 — Identify the 10 most specific, factual claims in the blog
STEP 3 — For each claim, run the Section 6 search protocol
STEP 4 — Keep only the claims verified on Tier 1 sources
STEP 5 — Select the best structure (A, B, or C) based on topic type
STEP 6 — Write the headline using the Section 5 formula
STEP 7 — Write 5-7 body items (verified claims only, 18 words max each)
STEP 8 — Write the curiosity gap (honest, points to real blog content)
STEP 9 — Write the CTA (specific value, blog URL)
STEP 10 — Run the full Section 7 checklist
STEP 11 — Output final pin copy in this format:

--- PIN COPY OUTPUT ---
STRUCTURE TYPE: [A / B / C]
HEADLINE: [exact text]
ITEMS:
① [exact text]
② [exact text]
③ [exact text]
④ [exact text]
⑤ [exact text]

CURIOSITY GAP: [exact text]
CTA: [exact text]
URL: [blog URL]
SOURCE BADGE: [Tier 1 source name to display on pin]
VERIFIED SOURCES: [list of Tier 1 URLs used — for your records]
CANVA NOTES: [any specific design instructions for this pin's layout]
------------------------

STEP 12 — Read every word of PIN COPY OUTPUT against your Tier 1 sources
STEP 13 — Pass the PIN COPY OUTPUT to Canva for design execution
```
